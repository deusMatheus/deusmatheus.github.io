Projetos

  • Landing Page

    Uma Landing Page (ou Página de Destino) é uma página da internet criada com um objetivo específico: converter visitantes em clientes ou interessados. Diferente de um site tradicional, que pode ter várias seções e informações, uma página de destino é direta e focada, geralmente com uma única chamada para ação, como:

    • Capturar e-mails para uma lista de contatos

    • Oferecer um livro digital ou material gratuito

    • Apresentar e vender um produto ou serviço

    • Inscrever usuários em um evento ou palestra online

  • Sistema de vendas de produtos

    Este é um sistema onde é possível realizar o cadastro de produtos para controle fincaneiro de vendas realizadas. Aliado à isso, há a possibilidade de cadastro de funcionários com login e senha próprio. O gestor deste sistema tem acesso às vendas de cada funcionário além de poder realizar o cadastro e exclusão dos produtos. Confira abaixo algumas imagens mostrando a interface básica do software com algumas funções, onde o design é totalmente personalizável!

  • Sistema de gestão de pedidos para restaurantes e pequenos negócios

    Este é um sistema onde é possível cadastrar pratos, lanches e pedidos dos clientes. Os clientes têm acesso somente ao cardápio que pode ser acessado através de QRCode. O software conta com sistema de login com diferentes acessos, como Caixa, Produção e Administração.

    • O usuário Caixa realiza os pedidos e recebe o pagamento dos clientes, informando o método de pagamento. Para o caso de pagamento em dinheiro, o sistema calcula o troco automaticamente!
    • O usuário Produção verifica os pedidos que estão na fila para serem produzidos. Este usuário pode alterar o estado dos pedidos de "Na fila" para "Em produção", e de "Em produção" para "Finalizado". Dessa forma, clientes podem verificar o estado através do Cardápio, informando o código do mesmo e obtendo assim a descrição com os itens e o estado atual.
    • O usuário Administração tem acesso às funções dos outros usuários além de poder cadastrar e excluir produtos. Ainda, é possível retirar relatórios de vendas determinando um período, para que possa verificar a quantidade de itens vendidos neste período, podendo organizar melhor seu negócio de acordo com a demanda.

  • Geração de gráficos

    Utilizando bibliotecas como MatPlotLib, NumPy e outras é possível gerar gráficos para diferentes cenários. Estes cenários incluem o plot de equações diferenciais, funções diversas e até mesmo gráficos baseados em dados coletados.

    As curvas abaixo representam a evolução dos parâmetros de densidade das componentes do universo de acordo com o redshift.   • Orders software system for restaurants and local business

    This system enables to register plates, snacks, fast food and clients orders. The clients can see the restaurant menu through a QRCode. The system have different levels of permission, the main beign the Cashier, Production and Management.

    • The Cashier user makes the client orders and recieves the payment, and can register the payment form (card, money...). If the payment is done by money, the system automatically calculates the change!
    • The Production user can see which orders are in queue. This user can alter a order status from "Queue" to "In Production", and from "In Production" to "Finished". By doing this, the clients can verify the order status by informing the order number at the Menu, getting the items description and current status.
    • The Management user has all the other users functionalities, and can also register and delete products from the menu. Finally, this user can generate reports by informing a data range, getting the number of items sold and all sales in given data.
